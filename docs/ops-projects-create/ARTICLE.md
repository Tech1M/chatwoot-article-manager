# Create a project

> Set up a project so timesheets, expenses, and milestones have a clear home and you can report on the work in one place.

## Before you start
- You know the project name, owner, and rough start and end dates.
- You have a budget figure or rate structure if you plan to track spend.
- You know which department or client the project belongs to.

## Steps

1. Open **Operations → Projects** and click **New project**.
   ![Projects page with new project button](./assets/01-projects-page.png)
2. Enter the project name, pick an icon and color, and set the project type (internal, client, or however your company uses it).
   ![New project form basic details](./assets/02-project-basics.png)
3. Add the project owner, start and end dates, and an optional budget. These fields drive reports later, so it is worth filling them in even if estimates change.
   ![Owner, dates and budget fields](./assets/03-project-owner-budget.png)
4. Save the project. You can now assign contractors, create milestones, and start logging time and expenses against it.

## Tips
- Use a consistent naming scheme (for example, `Client – Initiative – Year`) so projects are easy to find later.
- Close projects when work ends — closed projects keep their history but disappear from active dropdowns.

## Troubleshooting
- **Cannot create a project** — your role may not include project management; ask an admin to grant the permission.
- **Owner dropdown is empty** — only active staff can be project owners; invite the person first.
