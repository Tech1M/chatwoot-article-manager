# Set up a take-home assessment

> Take-home assignments give candidates space to do real work — design briefs, written analysis, code repos — on their own time.

## Before you start
- A written brief, any reference files, and a clear deadline window.
- Evaluation criteria so reviewers grade consistently.

## Steps

1. In the assessment builder, click **Add component → Take-home assignment**.
   ![Add take-home component](./assets/01-add-takehome.png)
2. Paste the brief, attach any reference files, and list what candidates must submit (PDF, link, repo, etc.). Add the evaluation rubric reviewers will use.
   ![Take-home brief and rubric](./assets/02-brief-rubric.png)
3. Set the **deadline** — either a fixed date or a duration that starts when the candidate opens the assignment. Choose whether late submissions are allowed.
   ![Deadline settings](./assets/03-deadline.png)
4. Save the component. Candidates will see the brief, upload their submission, and add notes for the reviewer.

## Tips
- Keep take-homes under 4 hours of work. Longer briefs hurt completion rates and shrink your candidate pool.
- Share the rubric in the brief — candidates appreciate it, and submissions come in better.

## Troubleshooting
- **Candidate can't upload a file** — check the allowed file types and the per-file size limit in the component settings.
- **Deadline passed but candidate hasn't started** — extend the deadline from the candidate's assessment session, or send a fresh invite.
