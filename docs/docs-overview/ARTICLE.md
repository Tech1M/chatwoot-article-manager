# Documents overview

> Use Documents to turn repeatable agreements into reusable templates and capture legally binding signatures inside the app.

## What you can do
- Build reusable templates for offers, contracts, NDAs, and policies.
- Pull data from your company, the contract, and the signer so you don't retype anything.
- Send documents to staff, talent, and external signers.
- Track who has signed, who is pending, and download a completion certificate when everyone is done.

## How it fits together

1. **Templates** hold the document body, the signer roles, and any merge variables.
   ![Template library](./assets/01-template-library.png)
2. **Documents** are generated from a template for a specific contract or recipient. Each document captures a snapshot of the template, so later edits to the template don't change documents already out for signature.
   ![Document detail](./assets/02-document-detail.png)
3. **Signatures** are collected in-app or through a secure link. Once everyone signs, you can download a completion certificate with the full audit trail.

## Tips
- Start with one template per document type and clone it as you discover edge cases.
- Use merge variables instead of hard-coded names and amounts so the same template works for every recipient.
- Set an expiry date on time-sensitive documents so stale links don't sit open.

## Troubleshooting
- **A signer can't open the document** — check that their email is correct and that the link hasn't expired.
- **Variables show empty values** — confirm the contract or recipient record has those fields filled in before generating.
